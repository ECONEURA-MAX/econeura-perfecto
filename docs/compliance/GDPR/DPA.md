# Data Processing Agreement (DPA)

**PLANTILLA - Requiere revisión legal**

## Partes

- **Controlador de datos:** Cliente
- **Procesador de datos:** ECONEURA

## Objeto del tratamiento

Procesamiento de datos del personal del Cliente para proporcionar servicios de automatización con IA.

## Naturaleza y finalidad

Ejecución de tareas automatizadas via NEURAs y agentes conectados (Make/n8n).

## Tipos de datos personales

- Datos identificativos: nombre, email
- Datos profesionales: cargo, departamento
- Logs de actividad: timestamps, acciones realizadas

## Categorías de interesados

Empleados del Cliente que usan la plataforma.

## Duración del tratamiento

Durante la vigencia del contrato + periodo de retención legal (ver [DATA_RETENTION.md](./DATA_RETENTION.md)).

## Obligaciones del procesador

1. Procesar datos solo según instrucciones del Controlador
2. Garantizar confidencialidad del personal
3. Implementar medidas técnicas y organizativas apropiadas
4. Asistir al Controlador en respuesta a ejercicio de derechos
5. Notificar brechas de seguridad en <72h
6. Eliminar/devolver datos al término del contrato

## Subprocesadores

- Azure (Microsoft) - hosting EU
- OpenAI - API (con DPA de OpenAI)
- Make.com - webhooks (con DPA de Make)

Cliente autoriza subprocesadores listados. Cambios notificados con 30 días antelación.

## Transferencias internacionales

Datos permanecen en UE (región North Europe). Si transferencia necesaria, se usarán cláusulas contractuales tipo de la Comisión Europea.

## Auditorías

Cliente puede auditar cumplimiento una vez al año, previa notificación de 30 días.
